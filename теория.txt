1. Ссылки в C++
Ссылка - это псевдоним для существующей переменной. После инициализации ссылка всегда refers to тот же объект.
Отличия от указателей:

Ссылка должна быть инициализирована при объявлении, указатель - нет

Ссылку нельзя переприсвоить на другой объект после инициализации

Ссылка не может быть nullptr, указатель - может

Для ссылок не нужно использовать & для доступа к значению
2. Константные ссылки
const T& - ссылка, которая не позволяет изменять объект.
3. Способы передачи аргументов в функцию
Когда что использовать:

По значению: для простых типов (int, double), когда изменения не нужны

По ссылке: когда нужно изменить оригинал или избежать копирования

По константной ссылке: для больших объектов, которые не нужно изменять

Через указатель: когда нужна возможность передавать nullptr или в C-style коде
4. Передача по значению
Да, создается копия объекта:
Накладные расходы: копирование большого объема данных может быть дорогим.

Когда оправдана:

Для простых типов (int, double, pointers)

Когда нужна локальная копия для модификации

Для небольших структур (до ~16-32 байт)
5. Передача больших структур по константной ссылке 
Преимущества:

Избегаем копирования большого объема данных

Экономия памяти и времени

Гарантия, что объект не будет изменен

6. Возврат ссылки на локальную переменную
Это опасно! Локальная переменная уничтожается при выходе из функции
Результат: неопределенное поведение (undefined behavior) - программа может работать неправильно или аварийно завершиться.

13. const_cast
const_cast используется для снятия или добавления константности.
Когда использовать const_cast:

Для вызова устаревших функций, которые не используют const

Когда точно известно, что объект не является константным, но передается через константную ссылку

Никогда для модификации действительно константных объектов
